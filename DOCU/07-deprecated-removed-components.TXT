QUEUE SYSTEM (src/common/queue/)
---------------------------------
All files in this directory have been deleted:

1. queue.module.ts
   - Purpose: NestJS module configuring Bull queue with Redis connection
   - Functionality:
     * Configured BullModule with Redis host/port from environment variables
     * Registered two queues: 'order-timeout' and 'kafupost-cleanup'
     * Set default job options (attempts, backoff, removeOnComplete/fail)
     * Imported TypeOrmModule for KafuPost entity
     * Imported EmergencyOrderModule for timeout processing
   - Dependencies: @nestjs/bull, @nestjs/config, @nestjs/typeorm
   - Redis Config: REDIS_HOST, REDIS_PORT (from .env)

2. order-timeout.processor.ts
   - Purpose: Bull processor for handling emergency order timeouts
   - Functionality:
     * Processed 'check-timeouts' jobs from 'order-timeout' queue
     * Called EmergencyOrderService.processOrderTimeouts()
     * Ran periodically via recurring job (every 60 seconds)
   - Dependencies: @nestjs/bull, EmergencyOrderService
   - Job Type: 'check-timeouts'

3. kafupost-cleanup.processor.ts
   - Purpose: Bull processor for deactivating expired KafuPosts
   - Functionality:
     * Processed 'deactivate-expired' jobs from 'kafupost-cleanup' queue
     * Queried database for posts where expiresAt < current time
     * Updated status from 'Open' to 'Cancelled' for expired posts
     * Ran daily via recurring job (every 24 hours)
   - Dependencies: @nestjs/bull, TypeORM, KafuPost entity
   - Job Type: 'deactivate-expired'

================================================================================
ALTERNATIVES / MIGRATION PATH
================================================================================

If queue-based background processing is needed in the future, consider:

Option 1: @nestjs/bullmq + bullmq (Redis alternative: not applicable)
  - Still requires Redis, but with modern BullMQ API
  - Migration: Reimplement processors with BullMQ syntax

Option 2: Cron Jobs (node-cron)
  - No external dependencies
  - Simpler API for scheduled tasks
  - Example: Use @nestjs/schedule module with cron expressions
  - Trade-off: No distributed queue, runs on single instance

Option 3: Database-Based Scheduler
  - Store scheduled tasks in database
  - Use setInterval or cron to poll and execute
  - More control, no Redis dependency
  - Trade-off: Slightly more complex implementation

For Push Notifications replacement:
  - Consider OneSignal, Expo Notifications, or custom WebSocket solution
  - Evaluate based on target platform (web, mobile, desktop)
