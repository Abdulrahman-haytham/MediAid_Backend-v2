## Emergency Order Scoring

### Algorithm (Max 100 points)

```
Total Score = Distance (0-50) + Rating (0-30) + Availability (0-20)
```

**Distance Score** (0-50 points):
```
score = GREATEST(0, 50 - (distance / 50000) * 50)
```
- 0m distance → 50 points
- 50km distance → 0 points
- >50km → Not included

**Rating Score** (0-30 points):
```
score = (averageRating / 5) * 30
```
- 5.0 rating → 30 points
- 2.5 rating → 15 points
- 0.0 rating → 0 points

**Availability Score** (0-20 points):
```
score = quantity > 0 ? 20 : 0
```
- In stock → 20 points
- Out of stock → 0 points

**Example**:
```
Pharmacy A:
- Distance: 5km → 45 points
- Rating: 4.5 → 27 points
- In stock → 20 points
- Total: 92 points

Pharmacy B:
- Distance: 25km → 25 points
- Rating: 3.0 → 18 points
- In stock → 20 points
- Total: 63 points
```

## Common Workflows

### User Journey: Browse & Order

```
1. Register → Verify Email → Login
2. Browse Categories → View Products
3. Search Nearby Pharmacies (PostGIS)
4. Add Items to Cart (with stock validation)
5. Create Order (transactional, deducts stock)
6. Track Order Status
7. Rate Order (when delivered)
```

### Pharmacist Journey: Manage Pharmacy

```
1. Register as Pharmacist → Verify → Login
2. Create Pharmacy (with location)
3. Add Products to Stock
4. Receive Orders
5. Update Order Status (pending → accepted → preparing → in_delivery → delivered)
6. Respond to Emergency Orders
7. Monitor Stock (expiring, low stock alerts)
```

### Emergency Order Journey

```
1. User Creates Emergency Order (with location)
2. System Scores Pharmacies (distance + rating + availability)
3. Top 5 Pharmacies Targeted
4. Pharmacies Respond (accept/reject)
5. Background Job Marks No Response if timeout
6. User Notified of Responses
```

## Troubleshooting

### Common Issues

**Issue**: Cannot login
- **Check**: Is email verified? (`isVerified: true`)
- **Check**: JWT_SECRET matches
- **Check**: Password correct (case-sensitive)

**Issue**: Order creation fails
- **Check**: Pharmacy exists and active
- **Check**: Stock sufficient
- **Check**: Delivery address provided (if delivery order)
- **Check**: Database connection stable

**Issue**: PostGIS queries fail
- **Check**: PostGIS extension enabled
- **Check**: Location columns created
- **Check**: Valid coordinates provided

**Issue**: Email not sending
- **Check**: SMTP credentials correct
- **Check**: App password (not account password) for Gmail
- **Check**: Port and TLS settings
- **Note**: Registration continues even if email fails (logged)

**Issue**: File upload fails
- **Check**: File size <= 5MB
- **Check**: File type is jpeg/png/jpg/gif
- **Check**: Cloudinary credentials correct

**Issue**: Emergency orders not matching
- **Check**: User location provided (lat/lng)
- **Check**: Pharmacies have location data
- **Check**: PostGIS spatial indexes exist
- **Check**: Pharmacies within 50km radius

## Glossary

| Term | Definition |
|------|------------|
| **JWT** | JSON Web Token - stateless authentication |
| **RBAC** | Role-Based Access Control |
| **PostGIS** | PostgreSQL extension for spatial data |
| **GIST** | Generalized Search Tree - index type for spatial data |

| **Entity** | Database model class in TypeORM |

| **Pipe** | NestJS data validator/transformer |
