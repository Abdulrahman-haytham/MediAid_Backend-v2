# Permissions & Authorization System

## Overview

The MediAid Backend implements a multi-layered authorization system combining:
1. **JWT-based Authentication**: Token-based identity verification
2. **Role-Based Access Control (RBAC)**: Three distinct user roles with specific permissions
3. **Resource Ownership Checks**: Fine-grained access control at the service level
4. **Endpoint-Level Guards**: Route-specific authorization via decorators

## Authentication Flow

### 1. Token Generation

```
User Registration → Email Verification → Login → JWT Token
```

**Step 1: Registration**
```
POST /users/register
{
  "email": "user@example.com",
  "password": "secure_password",
  "type": "user"
}
→ Creates user with isVerified: false
→ Generates 6-digit verification code
→ Sends verification email
```

**Step 2: Email Verification**
```
POST /auth/verify-email
{
  "email": "user@example.com",
  "code": "123456"
}
→ Validates code
→ Sets isVerified: true
```

**Step 3: Login**
```
POST /auth/login
{
  "email": "user@example.com",
  "password": "secure_password"
}
→ Validates credentials AND isVerified flag
→ Generates JWT token
```

### 2. JWT Token Structure

**Payload**:
```json
{
  "sub": "user-uuid-here",
  "email": "user@example.com",
  "role": "user",
  "iat": 1712912400,
  "exp": 1713776400
}
```

**Claims**:
- `sub` (Subject): User ID
- `email`: User email address
- `role`: User type (admin/user/pharmacist)
- `iat` (Issued At): Token creation timestamp
- `exp` (Expiration): Token expiration timestamp

**Configuration**:
- **Secret**: `JWT_SECRET` environment variable (REQUIRED, no weak fallback)
- **Expiration**: `JWT_EXPIRES_IN` environment variable (default: 7 days, configurable)
- **Algorithm**: HS256 (HMAC with SHA-256)

### 3. Token Validation

**JwtStrategy** (`src/modules/auth/jwt.strategy.ts`):

```typescript
@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor() {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: process.env.JWT_SECRET,
    });
  }

  async validate(payload: any) {
    // Load user from database by ID
    const user = await this.usersService.findOne(payload.sub);
    
    // Return user object (attached to req.user)
    return {
      id: user.id,
      email: user.email,
      type: user.type,
    };
  }
}
```

**Process**:
1. Extract Bearer token from `Authorization` header
2. Verify token signature and expiration
3. Decode payload
4. Load user from database
5. Attach user to `req.user`

## Role-Based Access Control (RBAC)

### User Roles

The system defines three distinct roles in the `UserRole` enum:

```typescript
export enum UserRole {
  ADMIN = 'admin',
  USER = 'user',
  PHARMACIST = 'pharmacist',
}
```

### Role Descriptions

| Role | Description | Access Level |
|------|-------------|--------------|
| **admin** | System administrators | Full access to all resources, administrative CRUD operations |
| **user** | Regular customers | Browse, purchase, manage personal data, community features |
| **pharmacist** | Pharmacy owners | Manage pharmacy, inventory, respond to orders |

### RolesGuard Implementation

**Guard** (`src/common/guards/roles.guard.ts`):

```typescript
@Injectable()
export class RolesGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    // Get required roles from @Roles() decorator metadata
    const requiredRoles = this.reflector.getAllAndOverride<UserRole[]>('roles', [
      context.getHandler(),
      context.getClass(),
    ]);
    
    // If no roles specified, allow access (if authenticated)
    if (!requiredRoles) {
      return true;
    }
    
    // Get user from request (attached by JwtStrategy)
    const request = context.switchToHttp().getRequest();
    const user = request.user;
    
    // Check if user role matches any required role
    return requiredRoles.some((role) => user.type === role);
  }
}
```

**Usage**:

```typescript
@UseGuards(AuthGuard('jwt'), RolesGuard)
@Roles(UserRole.ADMIN)
@Post('products')
createProduct(@Body() dto: CreateProductDto) {
  // Only admins can access
}
```

### @Roles Decorator

**Decorator** (`src/common/guards/roles.decorator.ts`):

```typescript
export function Roles(...roles: UserRole[]) {
  return SetMetadata('roles', roles);
}
```

**Usage Patterns**:

```typescript
// Single role
@Roles(UserRole.ADMIN)

// Multiple roles (OR logic)
@Roles(UserRole.ADMIN, UserRole.PHARMACIST)
```

### Level 4: Ownership-Based Access Control

Valid JWT + resource ownership verification.

**Examples**:
- `PATCH /users/:id` (update user - owner or admin)
- `DELETE /users/:id` (delete user - owner or admin)
- `GET /orders/:orderId` (get order - owner, pharmacist, or admin)
- `PUT /orders/:orderId/status` (update order status - role-based transitions)
- `DELETE /kafu-posts/:id` (delete post - owner or admin)

**Implementation** (Order Service):

```typescript
async findOne(orderId: string, user: User) {
  const order = await this.orderRepository.findOne({
    where: { id: orderId },
    relations: ['items', 'user', 'pharmacy'],
  });

  if (!order) {
    throw new NotFoundException('Order not found');
  }

  // Ownership check: user must be owner, pharmacy owner, or admin
  if (
    order.user.id !== user.id &&
    order.pharmacy.user.id !== user.id &&
    user.type !== UserRole.ADMIN
  ) {
    throw new ForbiddenException('Access denied');
  }

  return order;
}
```

**Implementation** (User Service):

```typescript
async update(userId: string, updateUserDto: UpdateUserDto, currentUser: User) {
  // Only owner or admin can update
  if (userId !== currentUser.id && currentUser.type !== UserRole.ADMIN) {
    throw new ForbiddenException('Access denied');
  }

  return this.userRepository.save({ id: userId, ...updateUserDto });
}
```

## Order Status Transition System

### Status Enum

```typescript
export enum OrderStatus {
  PENDING = 'pending',
  ACCEPTED = 'accepted',
  REJECTED = 'rejected',
  PREPARING = 'preparing',
  IN_DELIVERY = 'in_delivery',
  DELIVERED = 'delivered',
  CANCELED = 'canceled',
}
```

### Transition Rules by Role

The system implements a strict state machine with role-based transition permissions:

#### User (Customer) Permissions

| From Status | Can Transition To | Action |
|-------------|-------------------|--------|
| `pending` | `canceled` | Cancel own pending order |
| Other | ❌ No transitions | Users cannot modify other statuses |

**Implementation**:
```typescript
if (user.type === UserRole.USER) {
  if (order.status !== OrderStatus.PENDING) {
    throw new ForbiddenException('Users can only cancel pending orders');
  }
  newStatus = OrderStatus.CANCELED;
}
```

#### Pharmacist Permissions

| From Status | Can Transition To | Action |
|-------------|-------------------|--------|
| `pending` | `accepted`, `rejected`, `canceled` | Process incoming order |
| `accepted` | `preparing`, `in_delivery`, `canceled` | Advance order |
| `preparing` | `in_delivery`, `canceled` | Mark as preparing |
| `in_delivery` | `delivered`, `canceled` | Mark as in delivery |
| Other | ❌ No transitions | Cannot modify delivered/canceled orders |

**Implementation**:
```typescript
if (user.type === UserRole.PHARMACIST) {
  const allowedTransitions = {
    [OrderStatus.PENDING]: [OrderStatus.ACCEPTED, OrderStatus.REJECTED, OrderStatus.CANCELED],
    [OrderStatus.ACCEPTED]: [OrderStatus.PREPARING, OrderStatus.IN_DELIVERY, OrderStatus.CANCELED],
    [OrderStatus.PREPARING]: [OrderStatus.IN_DELIVERY, OrderStatus.CANCELED],
    [OrderStatus.IN_DELIVERY]: [OrderStatus.DELIVERED, OrderStatus.CANCELED],
  };

  if (!allowedTransitions[order.status]?.includes(newStatus)) {
    throw new ForbiddenException(`Invalid status transition from ${order.status} to ${newStatus}`);
  }
}
```

#### Admin Permissions

| From Status | Can Transition To | Action |
|-------------|-------------------|--------|
| `pending` | `accepted`, `rejected`, `preparing`, `in_delivery`, `delivered`, `canceled` | Full override |
| `accepted` | `preparing`, `in_delivery`, `delivered`, `canceled` | Advance order |
| `preparing` | `in_delivery`, `delivered`, `canceled` | Advance order |
| `in_delivery` | `delivered`, `canceled` | Complete or cancel |
| Other | ✅ Most transitions | Admins have broadest permissions |

**Implementation**:
```typescript
if (user.type === UserRole.ADMIN) {
  const allowedTransitions = {
    [OrderStatus.PENDING]: [
      OrderStatus.ACCEPTED,
      OrderStatus.REJECTED,
      OrderStatus.PREPARING,
      OrderStatus.IN_DELIVERY,
      OrderStatus.DELIVERED,
      OrderStatus.CANCELED,
    ],
    [OrderStatus.ACCEPTED]: [
      OrderStatus.PREPARING,
      OrderStatus.IN_DELIVERY,
      OrderStatus.DELIVERED,
      OrderStatus.CANCELED,
    ],
    [OrderStatus.PREPARING]: [OrderStatus.IN_DELIVERY, OrderStatus.DELIVERED, OrderStatus.CANCELED],
    [OrderStatus.IN_DELIVERY]: [OrderStatus.DELIVERED, OrderStatus.CANCELED],
  };

  if (!allowedTransitions[order.status]?.includes(newStatus)) {
    throw new ForbiddenException(`Invalid status transition`);
  }
}
```

### Inventory Restoration

When orders are canceled or rejected, inventory is automatically restored:

```typescript
async restoreOrderItemsStock(order: Order) {
  for (const item of order.items) {
    // Find pharmacy medicine with pessimistic lock
    const pharmacyMedicine = await this.pharmacyMedicineRepository.findOne({
      where: {
        pharmacy: { id: order.pharmacy.id },
        product: { id: item.product.id },
      },
      lock: { mode: 'pessimistic_write' },
    });

    if (pharmacyMedicine) {
      // Restore quantity
      pharmacyMedicine.quantity += item.quantity;
      await this.pharmacyMedicineRepository.save(pharmacyMedicine);
    }
  }
}
```

## Emergency Order Authorization

### Status Enum

```typescript
export enum EmergencyOrderStatus {
  PENDING = 'pending',
  ACCEPTED = 'accepted',
  FULFILLED = 'fulfilled',
  CANCELED = 'canceled',
  NO_RESPONSE = 'no_response',
}
```

### Response Permissions

| Action | Allowed Roles | Conditions |
|--------|---------------|------------|
| Create emergency order | Any authenticated user | Must provide location |
| View own emergency orders | Any authenticated user | Ownership check |
| Respond to emergency | Pharmacist only | Must be targeted pharmacy |
| Cancel emergency | Order owner | Only if pending |
| Fulfill emergency order | Order owner | Marks as fulfilled |

### Timeout Mechanism

**Background Job** (`src/common/jobs/order-timeout.job.ts`):

```typescript
@Injectable()
export class OrderTimeoutJob {
  private readonly CHECK_INTERVAL = 60000; // 60 seconds

  constructor(private emergencyOrderService: EmergencyOrderService) {
    // Run every 60 seconds
    setInterval(() => this.checkTimeouts(), this.CHECK_INTERVAL);
  }

  private async checkTimeouts() {
    const now = new Date();

    // Find expired pending orders
    const expiredOrders = await this.emergencyOrderRepository.find({
      where: {
        status: EmergencyOrderStatus.PENDING,
        responseTimeout: LessThan(now),
      },
    });

    // Mark as no_response
    for (const order of expiredOrders) {
      order.status = EmergencyOrderStatus.NO_RESPONSE;
      await this.emergencyOrderRepository.save(order);
    }
  }
}
```

**Timeout Configuration**:
- Default timeout: 15 minutes (`ORDER_CONSTANTS.DEFAULT_RESPONSE_TIMEOUT_MINUTES`)
- Check frequency: Every 60 seconds
- Expired status: `NO_RESPONSE`

### WebSocket Gateway Authorization

**Connection Authentication**:

```typescript
@WebSocketGateway({
  namespace: 'chat',
  cors: { origin: '*' },
})
export class ChatGateway {
  @WebSocketServer()
  server: Server;

  async handleConnection(client: Socket) {
    // Extract JWT from handshake
    const token = client.handshake.auth?.token?.replace('Bearer ', '');
    
    if (!token) {
      client.disconnect();
      return;
    }

    try {
      // Verify token
      const payload = this.jwtService.verify(token);
      client.data.user = payload;
      
      // Join user's personal room
      client.join(`user_${payload.sub}`);
    } catch (error) {
      client.disconnect();
    }
  }
}
```

**Order Room Access**:

```typescript
@SubscribeMessage('joinOrderRoom')
async handleJoinOrderRoom(client: Socket, payload: { orderId: string }) {
  const userId = client.data.user.sub;

  // Verify user has access to order
  const hasAccess = await this.orderService.hasOrderAccess(payload.orderId, userId);

  if (!hasAccess) {
    return { success: false, message: 'Access denied' };
  }

  // Join order room
  client.join(`order_${payload.orderId}`);
  return { success: true, message: 'Joined room' };
}
```

## Password Security

### Hashing Strategy

**Algorithm**: bcrypt
**Cost Factor**: 12 (2^12 = 4096 rounds)

**Hashing** (User Entity):

```typescript
@Entity()
export class User {
  @BeforeInsert()
  @BeforeUpdate()
  async hashPassword() {
    if (this.password) {
      this.password = await bcrypt.hash(this.password, 12);
    }
  }
}
```

**Verification** (Auth Service):

```typescript
async login(dto: LoginDto) {
  const user = await this.userService.findByEmail(dto.email);

  if (!user) {
    throw new UnauthorizedException('Invalid credentials');
  }

  const isPasswordValid = await bcrypt.compare(dto.password, user.password);

  if (!isPasswordValid) {
    throw new UnauthorizedException('Invalid credentials');
  }

  if (!user.isVerified) {
    throw new UnauthorizedException('Email not verified');
  }

  // Generate JWT token
  const payload = { sub: user.id, email: user.email, role: user.type };
  return {
    access_token: this.jwtService.sign(payload),
  };
}
```

### Password Reset Token System

**Token Generation**:

```typescript
async requestPasswordReset(dto: RequestPasswordResetDto) {
  const user = await this.userService.findByEmail(dto.email);

  // Generate random token
  const token = randomBytes(32).toString('hex');
  
  // Hash token before saving
  const hashedToken = createHash('sha256').update(token).digest('hex');

  user.resetPasswordToken = hashedToken;
  user.resetPasswordExpires = new Date(Date.now() + PASSWORD_RESET_EXPIRES);
  await this.userRepository.save(user);

  // Send email with unhashed token
  await this.mailService.sendPasswordResetEmail(user.email, token);
}
```

**Reset Verification**:

```typescript
async resetPassword(dto: ResetPasswordDto) {
  // Hash provided token
  const hashedToken = createHash('sha256').update(dto.token).digest('hex');

  const user = await this.userRepository.findOne({
    where: {
      resetPasswordToken: hashedToken,
      resetPasswordExpires: MoreThan(new Date()),
    },
  });

  if (!user) {
    throw new BadRequestException('Invalid or expired reset token');
  }

  user.password = dto.newPassword;
  user.resetPasswordToken = null;
  user.resetPasswordExpires = null;
  await this.userRepository.save(user);
}
```

**Token Expiration**:
- Default: 1 hour (3600000ms)
- Configurable via `PASSWORD_RESET_EXPIRES` environment variable

## Database-Level Security

### SQL Injection Prevention

**TypeORM Parameterization**:

All queries use parameterized statements, preventing SQL injection:

```typescript
// Safe - parameterized query
const user = await this.userRepository.findOne({
  where: { email: dto.email },
});

// Safe - raw SQL with parameters
const result = await this.pharmacyRepository.query(
  `SELECT * FROM pharmacies WHERE ST_DWithin(location::geography, $1::geography, $2)`,
  [userLocation, radius],
);
```

### Relationship Access Control

**Cascade Deletes**:

```typescript
@OneToMany(() => OrderItem, (item) => item.order, { cascade: true, onDelete: 'CASCADE' })
items: OrderItem[];
```

Ensures referential integrity - deleting an order automatically deletes its items.

**Eager Loading**:

```typescript
@ManyToOne(() => User, { eager: true })
user: User;
```

Critical relations loaded automatically to prevent unauthorized data access patterns.

## Common Security Patterns

### Pattern 1: Ownership Check

```typescript
// Always verify ownership before allowing access
async update(userId: string, dto: UpdateDto, currentUser: User) {
  if (userId !== currentUser.id && currentUser.type !== UserRole.ADMIN) {
    throw new ForbiddenException('Access denied');
  }
  // Proceed with update
}
```

### Pattern 2: Role-Based Filtering

```typescript
// Filter data based on user role
async findAll(user: User) {
  if (user.type === UserRole.PHARMACIST) {
    // Pharmacists see only their own + admin-created items
    return this.repository.find({
      where: [
        { createdBy: user, isAdminCreated: false },
        { isAdminCreated: true },
      ],
    });
  }
  // Others see all active items
  return this.repository.find({ where: { isActive: true } });
}
```

### Pattern 3: Conditional Validation

```typescript
// Validate based on context
if (order.status === OrderStatus.DELIVERED) {
  throw new BadRequestException('Order already delivered');
}

if (order.user.id !== user.id) {
  throw new ForbiddenException('Not your order');
}
```

### Pattern 4: Pessimistic Locking

```typescript
// Prevent race conditions on critical updates
const item = await this.repository.findOne({
  where: { id: itemId },
  lock: { mode: 'pessimistic_write' },
});

// Modify and save within transaction
item.quantity -= requestedQuantity;
await this.repository.save(item);
```
